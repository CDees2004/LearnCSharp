Properties: 
	Used in classes. Replacement for getters and setters. Has body 
	like a method but is referenced like a field. 
	
	// backing field 
	private int_count; 
	
	// property 
	public int Count {
		get { return_count; }
		set { _count = value; } // value is a keyword here 
		
	// auto implemented property 
	public int Count { get; set; } 
	
	// auto imp property with accessibility change 
	public int Count { get; private set; } 
	
	// can only go from more accessible to less accessible, 
	// not the other way around 
	// cannot do { public get;  set; } 
	
	// read only property 
	private int _count; 
	public int Count {
		get {return _count; }
	}
	
	// write only property 
	private int _count; 
	public int Count {
		set { _count = value; } 
	}
	
var keyword: 
	Variables declared with var still have compile-time types that 
	cannot change. 
	Variables declared with var must be initialized in the same 
	statement, so the type is known to the compiler. 
	Only valid on local variables. 
	
	var c = 'x'; // compile time char 
	c = 3; // compile-time error 
	
	var exists for anonymous types whose names are generated by 
	the compiler 
	
	var newObj = new {name="Chandler", age = 21}; 
	string myName = newObj.name; 
	
dynamic keyword: 
	Relieves compiler of determining the type. 
	Type is figured out at runtime. 
	If type doesn't match the methods/fields being references with
	it, then an exception is thrown. 
	You give up all compile time safety.
